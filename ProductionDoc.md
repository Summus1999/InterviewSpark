# 产品文档

## 1.产品介绍

这是一款面向个人求职者的Windows桌面应用，核心价值在于通过AI驱动的模拟面试和结构化复盘分析，帮助用户提升面试技巧、积累经验并发现改进方向。

## 2.核心功能规划

### 2.1 智能面试模拟

提供高度仿真的面试练习环境：

- 语音交互：实时语音识别，与AI面试官对话
- 个性化问题库：基于用户简历和岗位描述(JD)智能生成问题
- 多场景模式：支持技术面、行为面、压力面等不同场景

### 2.2 多维复盘分析

深度解析面试表现，提供具体改进建议:

- 内容分析：评估回答的逻辑性、与岗位的匹配度、关键词覆盖
- 表达分析：分析语速、停顿、清晰度
- AI洞见：基于模型分析指出优点、不足与优化方向

### 2.3 个人题库与成长追踪

沉淀面试经验，可视化进步历程：

- 个人题库：收藏高频问题与优化后的答案，支持标签分类管理
- 成长曲线：追踪多次模拟的评分变化，增强信心
- 活跃度热力图：GitHub 风格日历展示，直观查看练习频率

### 2.4 快速上手

降低使用门槛，提供开箱即用的体验：

- 预置模板：提供6个简历模板和6个JD模板，覆盖互联网常见职位（前端/后端/产品/全栈/QA/DevOps）
- 一键填充：点击选择模板，自动填入简历或JD内容，用户可在此基础上快速修改

### 2.5 个人画像与技能分析

提供多维度的能力分析，帮助用户深化自我认识：

- 个人画像：分析用户面试水平、优劣势、成长趋向等关键指标
- 行业对比：帮助用户了解自己在同业中的水平位置
- 智能练习推荐：基于薄弱点提出特定题干和技能学习路径
- 最佳实践：整合行业最佳实践和高质量答案示例

### 2.6 多用户支持

支持多用户数据隔离，方便家庭成员或多设备场景使用：

#### 2.6.1 用户创建

- 点击“+ 新建用户”按钮打开创建弹窗
- 输入用户名（必填）
- 选择或上传头像图片（支持 jpg/png 格式，类似微信头像）
- 点击“创建”完成

#### 2.6.2 用户编辑

- 点击用户卡片上的“编辑”按钮
- 可修改用户名
- 可更换头像图片
- 点击“保存”应用更改

#### 2.6.3 用户删除

- 点击用户卡片上的“删除”按钮
- 系统弹出确认提示（警告数据将被永久删除）
- 确认后删除用户及其所有数据（简历、面试记录、题库）
- 默认用户（ID=1）不可删除

#### 2.6.4 用户切换

- 点击用户列表中的任意用户卡片即可切换
- 切换后自动刷新页面，加载该用户的数据
- 当前用户以高亮边框标识

#### 2.6.5 使用引导

- 点击“查看使用引导”按钮可重新查看新手引导教程
- 重置所有提示气泡状态

#### 2.6.6 数据隔离

- 每个用户的简历、岗位描述、面试记录、题库完全独立
- 删除用户时级联删除其所有数据
- 无数据越权风险

### 2.7 数据管理

为保障用户数据安全，系统提供完善的数据管理功能：

#### 2.7.1 数据导出

- 功能入口：历史记录页面 → “备份数据”按钮
- 导出格式：JSON 全量导出
- 包含内容：
  - 所有用户信息
  - 全部面试会话记录
  - 题库收藏和标签
  - 简历和 JD 数据
- 使用场景：卸载前备份、转移数据、定期备份

#### 2.7.2 数据导入

- 功能入口：历史记录页面 → “恢复数据”按钮
- 导入源：之前导出的 JSON 备份文件
- 恢复策略：全量恢复，覆盖现有数据
- 使用场景：重新安装后恢复、跨设备同步

#### 2.7.3 卸载清理

- 自动清理：卸载应用时自动删除 AppData 目录下的所有用户数据
- 清理路径：`C:\Users\{用户名}\AppData\Roaming\com.interviewspark.app\`
- 包含内容：数据库文件、用户头像、所有配置文件
- 用户提醒：README 中明确说明卸载行为，提醒用户在卸载前备份重要数据

### 2.8 面试体验增强

提供更贴近真实面试的体验：

#### 2.8.1 计时模式

- 每道题独立计时，模拟真实面试时间压力
- 进度条实时显示剩余时间
- 超时警告提示（剩余 30 秒时颜色变化）
- 支持自定义每题时长（默认 3 分钟）

#### 2.8.2 AI 追问机制

- 基于用户答案自动生成追问
- 支持 5 种追问类型：澄清、深入、场景、挑战、拓展
- 用户可选择回答追问或跳过
- 记录对话历史，便于回顾

#### 2.8.3 STAR 法则评分

- 四维评估：Situation（情境）/ Task（任务）/ Action（行动）/ Result（结果）
- 雷达图可视化展示各维度得分
- 针对弱项提供改进建议

### 2.9 答案对比功能

支持同一问题不同时期的答案对比：

- 时间线模式：纵向排列历史回答，查看成长轨迹
- 并排对比模式：选择 2 个回答左右并列对比
- 显示评分变化：最早评分 → 最近评分
- AI 优秀答案参考：基于历史回答生成最佳答案

### 2.10 新手引导

降低使用门槛，提供引导式体验：

#### 2.10.1 首次使用引导

- 5 步交互式引导流程（欢迎/简历/JD/设置/开始）
- 可随时跳过，也可在设置中重新查看

#### 2.10.2 关键步骤提示

- 智能提示气泡引导操作
- 支持「不再显示」选项
- 四方向定位（上/下/左/右）

### 2.11 AI 反馈质量体系

提供高质量、可操作的 AI 反馈，帮助用户获得真正有价值的改进建议：

#### 2.11.1 结构化反馈展示

反馈内容采用分区卡片式布局，用户可快速定位关键信息：

- 评分区：圆环图展示综合评分 + 等级徽章（A+/A/B+/B/C+/C/D/F）
- 亮点区：绿色卡片展示回答优点，引用原文高亮显示
- 改进区：橙色卡片展示问题点，采用「问题 → 原因 → 修改示例」三段式结构
- 匹配度：标签云展示命中的岗位关键词 + 缺失要点提示

#### 2.11.2 面试官人设系统

提供多种面试官风格，满足不同练习场景需求：

- 严谨型（Strict）：高标准评估，直接指出问题，强调技术准确性和完整性
- 友好型（Friendly）：鼓励式反馈，先肯定优点再给建议，语气温和
- 压力型（Stress）：挑战性追问，质疑假设，测试候选人抗压能力
- 默认型（Default）：平衡客观的专业反馈

每种人设具备独特的评估侧重点、语气风格和典型措辞模式。

#### 2.11.3 行业特化评估

根据 JD 自动识别行业类型，加载对应评估维度：

- 技术岗：侧重技术深度、问题解决能力、系统设计思维
- 产品岗：侧重用户思维、数据驱动、跨团队沟通协作
- 运营岗：侧重执行力、复盘分析能力、资源整合能力

#### 2.11.4 反馈一致性保障

确保评分准确、反馈格式稳定：

- 评分校准：AI 评分与本地算法加权融合（AI 70% + 本地 30%），避免评分漂移
- 格式验证：自动检测 AI 输出是否符合预期结构，缺失字段时触发重试
- 失败降级：格式异常时自动重试（最多 2 次），仍失败则回退到简化版反馈

#### 2.11.5 反馈进化机制

持续优化反馈质量，形成正向循环：

- 反馈评价：每条反馈底部提供「有帮助 / 没帮助」评价按钮
- 数据收集：记录用户评价，标记低质量反馈样本
- 迭代优化：定期分析低评价反馈，针对性调整 Prompt 策略

## 3.技术选型

### 3.1 应用框架与开发语言

- **应用框架**：Tauri - 轻量级跨平台桌面应用框架，打包体积小、内存占用低
- **后端语言**：Rust - 高性能、内存安全，适合处理语音数据和本地业务逻辑
- **前端技术**：Vue 3 + TypeScript - 渐进式框架，开发效率高，组件化开发体验好

### 3.2 AI能力集成

- **大模型API**：硅基流动 - 提供面试问题生成、回答分析、复盘建议等核心AI能力，支持SSE流式输出
- **多模型支持**：Qwen3-8B / Qwen Plus / Qwen Max / Kimi Large / GLM-4-6v / MiniMax-M2
- **语音识别**：Web Speech API - 浏览器原生语音识别，支持中文识别，无需模型下载
- **语音合成**：Web Speech API - 浏览器原生TTS，音质自然，支持语速和音量调节

**技术方案说明**：

- 原计划使用 SenseVoice（本地）+ Piper TTS（本地）方案
- 实际采用 Web Speech API 方案，优势：
  - 轻量级，无需下载大模型文件
  - 跨平台兼容性好（Chrome/Edge推荐）
  - 开发和维护成本低
- 支持用户自定义模型和 API Key 配置

### 3.3 数据存储

- **本地数据库**：SQLite - 嵌入式数据库，无需额外部署，存储用户简历、面试记录、题库等数据
- **状态管理**：Pinia - Vue 3 状态管理库，实现题库和会话缓存
- **图表展示**：ECharts - 数据可视化图表库，用于雷达图、趋势图等展示
- **草稿存储**：IndexedDB - 本地数据库，用于答案草稿自动保存，防止数据丢失

## 4.开发规划线路

### 4.1 最小可行产品

**目标**：实现核心闭环。支持手动输入简历文本和JD，生成文本面试问题，用户输入文字答案，AI进行简单的文本分析和反馈。
**技术焦点**：完成Tauri + Rust基础框架搭建，成功集成硅基流动API进行文本交互。

### 4.2 增强部分功能体验

**目标**：加入语音识别和合成功能，实现真正的对话式模拟面试。完善复盘报告的可视化展示。
**技术焦点**：集成语音处理模块，优化UI和交互流畅度。
